## 🦚 git branch 전략

```bash
main: release 전용
dev: 개발 서버 배포 전용
feat: 기능 개발
fix: 버그 수정
refactor: 리팩토링
chore: 기타 작업
docs: 문서 작업
test: 테스트 코드 구현
```

- 각 브랜치는 dev 브랜치에서 생성
- 작업 후 dev 브랜치로 PR 후 merge
